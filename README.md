# pages-official-home

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存储，不绑定特定域名或网站，主要作为通用页面存档与分发使用。

## 项目简介

`pages-official-home` 是一个轻量级的 HTML 页面归档仓库。仓库内所有文件均为纯静态 HTML，无后端依赖，可直接通过 GitHub Pages 或其他静态托管服务浏览。适用于存放示例页面、存档页面、临时页面或无需动态功能的展示页面。

## 目录结构

```
pages-official-home/
├── index.html          # 仓库首页（如有）
├── page-1/             # 示例页面目录
│   ├── index.html
│   └── assets/
├── page-2/
│   ├── index.html
│   └── assets/
└── README.md           # 本文件
```

每个页面目录通常包含一个 `index.html` 及必要的资源文件（如 CSS、JavaScript、图片等）。目录命名建议使用有意义的英文或拼音，避免特殊字符。

## 页面归档说明

- 所有页面均为独立 HTML 文件，互不依赖。
- 页面资源（样式、脚本、图片）建议使用相对路径，确保目录移动后仍可正常访问。
- 如需引用外部资源（如 CDN），请确保链接长期有效。
- 归档页面应避免包含敏感信息、隐私数据或动态后端接口。

## 维护说明

- 添加新页面：在仓库根目录下新建文件夹，放入 `index.html` 及相关资源。
- 更新页面：直接修改对应目录下的文件。
- 删除页面：移除整个目录及其中所有文件。
- 提交时请附带简洁的提交信息，说明新增、修改或删除的内容。
- 建议保持目录结构整洁，避免堆放过多无关文件。

## 使用方式

1. 克隆或下载本仓库。
2. 在本地编辑 HTML 文件后提交推送。
3. 启用 GitHub Pages（Settings > Pages > Source 选择 main 分支）即可在线访问。
4. 也可将文件部署至任意静态服务器。

## 注意事项

- 本仓库不提供动态功能，所有页面均为纯静态。
- 不保证页面在所有浏览器或设备上完全兼容，请自行测试。
- 请遵守相关法律法规，勿上传违规内容。
